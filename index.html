Software Design and Engineering

For this milestone, I improved the application by cleaning up the code in a few ways: Firstly, I reorganized some of the methods and members to be more intuitive (e.g., moving “create” methods above “read” or “delete” methods to follow the CRUD model). Additionally, I provided more comprehensive comments to my code to explain the logical flow of the program in simple terms. Finally, I removed a number of unused imports to cut down on clutter.

For this artifact, the course outcomes were achieved. I made the improvements that I detailed in my original enhancement plan, and demonstrated my knowledge of the software development process. These improvements reflect software engineering fundamentals for building collaborative environments by providing helpful comments inside the code to act as guidelines for any future modifications. If another party were to review or modify this code, the comments I've provided would allow them to understand the program logic and flow more quickly, and identify the expected outcomes and return values for each method.

Possible further updates would include removing console logs and test methods to further reduce the number of lines of code. This milestone demonstrated to me that even a simple application can sometimes be challenging to design and build. It is important to focus on the core functionality when developing an app, however, it can be easy to fall victim to scope creep even early on in the development process as you do incremental testing. It was also helpful to revisit my work after stepping away for some time as it allowed me to look at my code from the perspective of a third party who would not be as familiar with my aims.

E2 Repository

Databases

For this milestone, I've connected the app to a Firebase Firestore instance in order to improve the database functionality. Before, the app was simply storing the data locally on the user's device. This allowed for extremely fast fetching and storage, however, it also meant that if anything happened to the user's device or that data, it would be lost forever. Storing the data on a cloud server requires quite a bit more overhead but also keeps it much safer. If the user changes devices or logs in from a different mobile phone, for example, their data will still be accessible.

This task showed me how difficult it can be to add database functionality to an existing app. Even though this particular app is comparitively very simple, it was still quite a challenge to switch from local storage to utilize the Firebase services from Google. It did not help much that Google's official documentation seems to be outdated. In the future, I will probably look into other solutions, although once everything was finally in place and working, I was happy with the results.
